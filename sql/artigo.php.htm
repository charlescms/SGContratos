Manipulando datas com a função EXTRACT

Manipulando datas com a função EXTRACT

Data da última atualização: 28/02/2011

O Firebird suporta três tipos de dados utilizados para armazenar data e/ou hora:

  • Timestamp: Armazena data e hora, conjuntamente.
  • Date: Armazena somente data.
  • Time: Armazena somente hora.

Obs: Esse artigo considera que o banco de dados está usando o Dialeto 3 e a versão do Firebird seja a 2.1 ou superior.

É comum, especialmente na filtragem de dados ou em queries utilizadas para geração de relatórios, utilizarmos a cláusula where para limitar os dados retornados, baseados em um intervalo de tempo. Sendo assim, se precisarmos recuperar a lista de notas fiscais emitidas no mês de Janeiro de 2.011, podemos fazer algo do tipo:

select * from notas
where (emissao >=  '1.1.2011 ') and (emissao <=  '2011-01-31');

ou

select * from notas
where (emissao between  '1.1.2011 ' and    '01/31/2011 ');

Dica: No Firebird, utilizamos “.” (ponto) para referenciar datas no formato dia/mês/ano. Se usarmos “/”, o Firebird interpretará como sendo mês/dia/ano. Se usarmos “-“, o formato esperado será ano/mês/dia.

Simples, rápido (especialmente se existir um índice para o campo emissao) e indolor. A coisa pode se complicar em casos não tão tradicionais, por exemplo: suponha que desejamos recuperar a lista dos clientes cujo aniversário (armazenado no campo aniversario do tipo date) seja em Fevereiro, independente do dia ou do ano.

Nesse caso, fica praticamente impossível  montar a query de forma similar aos exemplos anteriores. No entanto, graças a função EXTRACT do Firebird, o processo é ainda mais simples: 

select * from clientes
where extract(month from  aniversario) = 2;

No exemplo acima, a função EXTRACT foi usada para extrair o mês da data, comparando-o com “2” (Janeiro = 1, Fevereiro = 2, etc...). Na tabela abaixo podemos ver todas as variantes que a função EXTRACT aceita, bem como o tipo de dado retornado por cada uma:

Parâmetro Tipo retornado Faixa Comentário
YEAR SMALLINT 1–9999  
MONTH SMALLINT 1–12  
WEEK SMALLINT 1–53  
DAY SMALLINT 1–31  
WEEKDAY SMALLINT 0–6 0 = Domingo, 1 = Segunda, etc.
YEARDAY SMALLINT 0–365 0 = 1 de Janeiro
HOUR SMALLINT 0–23  
MINUTE SMALLINT 0–59  
SECOND NUMERIC(9,4) 0.0000–59.9999 A parte fracionária representa os milissegundos
MILLISECOND NUMERIC(9,1) 0.0000–999.9 * quebrado no FB 2.1 e 2.1.1

Obviamente, YEAR, MONTH, WEEK, DAY, WEEKDAY, YEARDAY só podem ser utilizados com TIMESTAMP ou DATE, visto que o tipo TIME armazena somente informações de hora. Conseqüentemente,  HOUR, MINUTE, SECOND e MILLISECOND só podem ser usados com dados do tipo TIMESTAMP e TIME.

Segue alguns exemplos descrevendo o uso de cada uma das opções do EXTRACT: YEAR:

select * from clientes
where extract(year from data_cadastro) = 2010;

Lista todos os clientes cadastrados no ano de 2.010.

WEEK: 

select * from notas
where (extract(week from emissao)  between 1 and 4) and
      (extract(year from emissao) =  2011);

Lista todas as notas fiscais emitidas nas quatro primeiras semanas do ano de 2.011.

Importante: O Firebird implementa o WEEK de acordo com a ISO-8601, que determina que as semanas começam na segunda-feira, e sempre contém sete dias completos. A semana 1 é a primeira semana que possuir a maior parte dos dias (pelo menos 4) do novo ano. Os primeiros “1 a 3” dias do novo ano podem pertencer a última semana (52 ou 53) do ano anterior. Da mesma forma, os últimos dias de um ano podem pertencer à primeira semana do ano seguinte.

DAY

select * from clientes
where extract(month from aniversario) = 1
order by extract(day from aniversario);

Lista todos os clientes que fazem aniversário em Janeiro, ordenados pelo dia do aniversário.

WEEKDAY

select * from contas_a_receber
where extract(weekday from vencimento) in (6,0);

Lista todos os lançamentos da tabela de Contas a Receber cujo vencimento ocorreu em um final de semana (sábado ou domingo).

YEARDAY

select * from clientes
where (extract(yearday from data_cadastro) <= 180) and
(extract(year from emissao) =  2011);

Lista todos os clientes cadastrados nos primeiros 180 dias do ano de 2.011.

HOUR

select * from log_acesso where extract(hour from data_acesso) between 0 and 6;

Lista os acessos logados entre meia-noite e 6 horas da manhã, independente da data.

MINUTE / SECOND / MILLISECOND

update clientes set hora_cadastro  = 
  cast(extract(hour from hora_cadastro) ||  ':  ' ||
  extract(minute from hora_cadastro)  ||  ':  ' ||
  extract(second from hora_cadastro)  ||  ':'  as TIME)
where extract(millisecond from hora_cadastro) <> 0;

O comando acima atualiza o campo hora_cadastro da tabela de clientes, removendo os milissegundos da informação.

Conclusão A função EXTRACT permite separar todos os elementos que compõem datas e horas, facilitando muito o trabalho de extração de informação. Além disso, pode ser usada em regras de validação (check constraints), índices, ordenação e campos calculados, bem como em qualquer comando sql, como selects, updates, inserts, etc.
